### 04.03.2025

**Информация** - любые сведения о каком-либо событии, объекте или процессе, являющиеся объектом некоторых операций: восприятия, передачи, преобразования, хранения, или использования. 

**Данные** - это информация, зафиксированная в некоторой форме ан некотором носителе, пригодной для последующей обработки, передачи и хранения, например, находящаяся в памяти ЭВМ или подготовленная для ввода в ЭВМ. 

**Обработка данных** - это совокупность хадач, осуществляющих преобразование массивов данных:
- ввод данных ЭВМ;
- отбор данных по каким либо критериям;
- преобразование структуры данных;
- перемещение данных на внешней памяти ЭВМ;
- вывод данных, являющихся результатом решения задач, в табличном или в каком либо ином удобном для пользователя виде. 

**Две основные предпосылки появления баз данных:**
1. Необходимость хранить и обрабатывать большое количество данных.
2. Разработка методов совместного использования данных.

**Система управления базами данных (СУБД, англ. Database Managment System)** - это комплекс программ для создания баз данных и управления данными. 

**Банк данных** - это система специальным образом организованных данных (баз данных), программных, технических, языковых, организационно-методических средств, предназначенных для обеспечения централизованного накопления и коллективного многоцелевого использования данных. 

**Типы баз данных**

**Клиент серверные** - доступ к данным осуществляется путем запроса к серверу БД, который и осуществляет непосредственные операции с данными. 
**Плюсы:** возможность работы с одной БД любого количества пользователей, минимизация передаваемого по сети трафика. 
**Минусы:** требуется отдельный сервер для полноценного функционирования. 

**Файл-серверные** - Данные хранятся на файловом сервере, доступ к ним осуществляют клиенты БД, установленные на каждом локальном компьютере. 

**Встраиваемые БД** - работа с даннымми осуществляется путем подключения к программе библиотек, реализующих работу с БД и передачи требуемых вызовов данным библиотекам. 
**Плюсы** - скорость работы (при относительно небольших объемах данных) и компактность системы. 
**Минусы** - невозможно масштабирование, катастрофическое падение производительности на крупных выборках. 

**Модель данных** - это совокупность правил порождения структур данных в базе данных, операций над ними, а также ограниченной целостности, определяющих допустимые связи и значения данных, последовательность их изменения [ГОСТ 20886-85].

**Модель данных состоит из трех частей:**
1. Набор типов структур данных;
2. Набор операторов или правил вывода;
3. Набор общих правил целостности. 

**Модели БД:**
1. **Иерархические** - состоит из элементов, расположенных в порядке подчинения (по типу дерева). 
2. **Сетевые** - это неупорядоченный набор веерных отношений (потомок-предок) (связь всех информационных объектов со всеми). 
3. **Реляционные** - (от латинского realtio - отношение) состоит из взаимосвязанных таблиц.

**Отношение в базе данных** - это структура, которая состоит из набора атрибутов (столбцов) и набора кортежей (строк). Оно представляет собой логическую таблицу, содержащую информацию об определенной сущности или объекте. Отношения используются для хранения, обработки и организации данных.

**Атрибуты отношения** - представляют собой свойства или характеристики объекта, например, имя, возраст, пол итд. 
**Кортежи** - содержат конкретные значения атрибутов (домен) для определенной записи в отношении. Отношения в базе данных связаны между собой через общие атрибуты, что позволяет объединять, сортировать и анализировать данные из разных таблиц. 

**Ключ** - идентификатор, являющийся частью набора элементов данных. В контексте реляционных СУБД ключом считается совокупность атрибутов отношения (или отдельный атрибут), обладающий определенными, характерными для данного вида ключа свойствами. 

**Типы ключей:**
1. **Первичный ключ** - это потенциальный ключ, который выбран для уникальной идентификации кортежей внутри отношения. 
2. **Внешний ключ** - это атрибут или множество атрибутов одного отношения, которые принимают значения потенциального ключа другого отношения (может быть и того же).

**Целостность** - свойство БД, означающее, что она содержит полную, непротиворечивую и адекватно отражающую предметную область информацию. 

**Физическая целостность** - сохранность данных в случае отказа оборудования. 
- транзакции;
- резервное копирование;
- RAID-массивы жестких дисков (дублирование).

**Логическая целостность** - непротиворечивость. 
- типы полей;
- обязательные поля;
- уникальные индексы;
- ограничения на значения;
- шаблоны ввода (###) ###-##-##

**Ограничения целостности** - правила, определяющие возможные значения в БД. 
- **категорийная** целостность - никакой атрибут первичного ключа не может быть пустым
- **ссылочная целостность** - каждой записи основной таблицы соответствует нуль или более записей дополнительной таблицы; каждая запись дополнительной таблицы имеет только одну родительскую запись основной таблицы. 

**Отношение один-ко-многим**
В отношении один-ко-многим каждой записи в одной таблице могут соответствовать несколько записей в другой таблице, но запись в другой таблице не может иметь более одной соответствующей ей записи в первой таблице. 

**Отношение один-к-одному** 
При отношении один-к-одному запись в первой таблице может иметь не более одной связанной записи в другой таблице и наоборот. 

**Отношение многие-ко-многим** 
При отношении многие-ко-многим одной записи в первой таблице могут соответствовать несколько записей во второй таблице, а одной записи во второй таблице несколько записей в первой таблице. Этот тип связи возможен только с помощью третьей (связующей) таблицы, первичный ключ которой состоит из двух полей, которые являются внешними ключами первой и второй таблиц. 

**Поле базы данных** - это столбец таблицы, содержащий значения определенного свойства. Каждое поле характеризуется своим именем и типом данных, представляющих значения данного свойства. 

**Запись базы данных** - это строка таблицы, содержащая набор значений свойств, размещенный в полях базы данных. 

**Запрос** - это средство выбора необходимой информации из БД. Вопрос, сформированный по отношению к базе данных, и есть запрос. 

**Виды запросов:**
- запрос на выборку
- запрос с параметрами
- итоговые запросы
- запрос с вычисляемыми полями
- запрос на обновление
- запрос на удаление
- запрос перекрестный
